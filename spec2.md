| **uniqueUserId / vectorHash**                           | Unique secure hash used to tag a user’s vector embeddings so that the “Lobotomy” script can locate and permanently delete them.                                                                                     | Should be cryptographically random; used as Firestore document ID and vector DB tag. |
| **username (string)**                                   | Core account data includes a username that is collected to save progress.  Admins, developers and employees can see each other’s usernames, but testers and ordinary users should not see any usernames.            |                                                                                      |
| **hashedEmail (string)**                                | Only a hashed/encrypted version of the user’s email is stored.  Real emails are never exposed to moderators.                                                                                                        |                                                                                      |
| **role (string)**                                       | Role of the account: `admin`, `dev`, `employee`, `tester`, or `user`.  Determines who can see usernames (per user’s note).                                                                                          |                                                                                      |
| **displayName (string)**                                | Public display name shown to others.  Adults can choose their own display name; minors must pick from a pre‑generated list (see minor fields).                                                                      |                                                                                      |
| **age / dateOfBirth (timestamp) and isMinor (boolean)** | Needed to determine whether the account is a minor (<18) and whether verifiable parental consent (VPC) is required.                                                                                                 |                                                                                      |
| **consentAOSModeration (boolean)**                      | Users must accept that their interactions are monitored by AI safety agents and have the right to request an AOS trace review.  Store `true` once the user scrolls to the bottom of the Informed‑Consent Agreement. |                                                                                      |
| **persistentAiMemory (boolean)**                        | Optional.  User may opt‑in to allow the AI to remember specific preferences; default off.                                                                                                                           |                                                                                      |
| **aggregateTrainingOptIn (boolean)**                    | Optional.  User may allow de‑identified data to be used in Tier‑3 differential‑privacy training.                                                                                                                    |                                                                                      |
| **neuralDataOptIn (boolean)**                           | Optional.  Users (or parents for minors) must explicitly opt‑in for adaptive gameplay based on sentiment analysis; if not selected, AI follows standard safety rails.                                               |                                                                                      |
| **city (string)** and **state (string)**                | Only coarse geolocation (city/state) is collected; no GPS or street‑level data.                                                                                                                                     |                                                                                      |
| **gameProgress (object) / achievements (array)**        | Stores information about game progress and achievements as part of “Gameplay Patterns”.                                                                                                                             |                                                                                      |
| **anonymizedAlias (string)**                            | Internal alias used for moderation and blind audits.  Moderators must never see real names, emails or IP addresses; users are represented as “Subject Alpha,” “Subject Beta,” etc.                                  |                                                                                      |
| **safetyLogsRef (reference/subcollection)**             | AI‑driven chat monitoring logs that track safety incidents (bullying, grooming, etc.).  These should be stored in a subcollection (`/users/{userId}/safetyLogs`) to avoid bloating the main user document.          |                                                                                      |
| **rightsRequestsRef (reference/subcollection)**         | Track user/parent requests for access, deletion or a “Lobotomy” in compliance with CTDPA rights.                                                                                                                    |                                                                                      |
| **lobotomyTimestamp (timestamp)**                       | Date/time when the user or parent requested the Lobotomy protocol, so the system knows that vector embeddings must be purged.                                                                                       |                                                                                      |
| **createdAt / updatedAt (timestamps)**                  | Standard Firestore timestamps for auditing.                                                                                                                                                                         |                                                                                      |
